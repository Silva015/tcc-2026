\label{chap:methodology}

This chapter details the data acquisition, the architectural implementation of the TransferKAN model, and the rigorous experimental setup used to train and validate the proposed architecture. The methodology is meticulously designed to ensure strict data separation, completely preventing the semantic data leakage that often plagues medical image classification tasks.

\section{Dataset Description}
\label{sec:dataset_desc}

The dataset utilized in this study is the FDCPUnBGunshotDB~\cite{Lira2025deep}, a curated collection of real crime scene and autopsy photographs obtained by the Federal District Civil Police (PCDF) in Brazil. Unlike datasets composed of staged laboratory environments or animal models, this dataset authentically reflects the severe class imbalances, lighting variations, and morphological distortions naturally occurring in forensic fieldwork.

For the primary \textbf{Wound Type Classification} task (Entry vs. Exit), the dataset contains a total of 2,551 images. This is severely imbalanced, comprising:
\begin{itemize}
    \item \textbf{Entry Wounds (Label 0):} 1,883 images
    \item \textbf{Exit Wounds (Label 1):} 668 images
\end{itemize}

For the secondary \textbf{Medical-Legal Shooting Distance (MLSD) Classification} task, only the 1,883 entry wound images are considered, as exit wounds do not display distance characteristics. These are categorized into three distinct, highly skewed classes:
\begin{itemize}
    \item \textbf{Distant (Label 2):} 1,609 images (85.45\%)
    \item \textbf{Close range (Label 1):} 232 images (12.32\%)
    \item \textbf{Contact (Label 0):} 42 images (2.23\%)
\end{itemize}

Beyond the class imbalance, the dataset reflects the demographic profile typical of firearm-related fatalities: the victims are predominantly young adult males, with a mean age of approximately 27 years and a pronounced right skew towards individuals in their twenties, as shown in Figure~\ref{fig:age_dist}.

\section{TransferKAN Implementation Details}
\label{sec:kan_implementation}

The core of our proposed solution is the \textbf{TransferKAN} architecture, implemented using PyTorch~\cite{paszke2019pytorch}.

\subsection{Backbone and Regularization}
We employed a pre-trained ResNet152~\cite{he2016resnet} from TorchVision as the backbone for spatial feature extraction, leveraging its pre-existing weights to identify complex low-level visual patterns without requiring a massive dataset. The standard final fully connected layer was stripped, leaving the network to output a high-dimensional flattened feature vector of size 2048. To combat overfitting on the limited dataset, a heavy Dropout layer ($p=0.5$) was inserted immediately after the feature extractor.

\subsection{KAN Classifier Configuration}
Instead of conventional dense layers, we appended a dynamic KAN module~\cite{liu2024kan} (using the \texttt{efficient-kan} implementation) as the final classifier.
For the Entry/Exit classification, the KAN was configured with a structure of [2048, 128, 2]. To ensure the splines could map the complex decision boundaries of the imbalanced data, we used a cubic spline order of 3 and an increased grid size of 10. For the MLSD classification, the structure was slightly adapted to [2048, 128, 3] to accommodate the three distance classes.

\subsection{Data Augmentation}
Due to the scarcity and extreme imbalance of forensic data, advanced Data Augmentation was applied exclusively to the training set using the \textit{Albumentations} library~\cite{buslaev2020albumentations}. The augmentation pipeline included transformations specifically tailored to mimic forensic realities:
\begin{itemize}
    \item \textbf{Geometric Transformations:} Horizontal and vertical flips, and shift-scale-rotate combinations to simulate different photographic angles.
    \item \textbf{Tissue Simulation:} Elastic transformations to simulate the natural elasticity and distortion of human skin.
    \item \textbf{Lighting and Detail:} Contrast-Limited Adaptive Histogram Equalization (CLAHE) to enhance tissue details, and Gaussian noise to simulate the varied, often poor lighting conditions found at crime scenes.
\end{itemize}
Figure~\ref{fig:augmentation} shows representative examples of these transformations applied to wound images of each type. The testing set underwent strictly deterministic transformations, restricted to essential resizing ($224 \times 224$) and normalization.

\subsection{Data Splitting: Preventing Semantic Data Leakage}
A common flaw in forensic machine learning is performing a random split of images. Since a single victim (Case ID) might have multiple photographs of the same wound taken from slightly different angles, randomly splitting images can result in the same wound appearing in both the training and testing sets, causing semantic data leakage and artificially inflating test metrics. 

To completely prevent this, the dataset was split strictly based on the \textit{Case ID} (extracted from the filename). We implemented a rigorous 5-fold cross-validation strategy, maintaining an 80\%/20\% case-based split per fold. A programmatic sanity check was utilized to guarantee zero semantic intersection between the train and test subsets.

\subsection{Two-Phase Training Procedure}
The training was conducted over 60 epochs per fold, utilizing a strict two-phase transfer learning methodology:
\begin{enumerate}
    \item \textbf{Phase 1 (Epochs 1-15):} The ResNet152 backbone parameters were frozen. Only the KAN classifier was trained using the AdamW optimizer with a learning rate of $1 \times 10^{-3}$ and weight decay ($1 \times 10^{-2}$). This allowed the KAN splines to converge to a stable baseline utilizing the raw features.
    \item \textbf{Phase 2 (Epochs 16-60):} The entire network was unfrozen (Fine-tuning). Training continued with a reduced learning rate of $1 \times 10^{-5}$ to jointly optimize the convolutional filters and the KAN splines.
\end{enumerate}
A CosineAnnealingLR scheduler was utilized throughout both phases to smoothly decay the learning rate. Furthermore, inverse class frequency weighting was applied to the Cross-Entropy Loss function to mathematically penalize errors in the minority classes (e.g., Contact shots) more heavily. During each fold, the model weights achieving the highest test accuracy were checkpointed, and the out-of-fold predictions were aggregated to calculate the final, true cross-validated metrics.
